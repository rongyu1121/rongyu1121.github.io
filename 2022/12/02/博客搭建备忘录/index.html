    摘要

    从2016年开始搭建博客，期间换过电脑，换过主题，到这次是第三次折腾博客了。每次弄都会有一些坑点，因此建此备忘录，以备将来需要时查看。本次搭建博客参照 这篇文章 。

    坑点
    • 添加 SSH key 并添加 github 账号设置里。
    • 本地和远程仓库建立关联后，推送代码 git push origin main ，注意现在默认仓库名是 main 而不是 master 了，这是一个坑点。
    • 安装 hexo 时命令 npm install -g hexo-cli 前需要加 sudo (mac电脑)
    • _config.yml 文件里 deploy: 部署的地方，branch 要写 main 不写会默认创建 master 分支，这会导致用 hexo d 命令部署时，又帮你创建 master 分支。
    • _config.yml 文件里 deploy: 部署的地方，仓库地址要写 ssh 的那个，写 https 的那个会导致推送的时候，让你输入账号密码，这种方式在 2021.08.31 就已经不准使用了。
    • 如果部署完主题，显示有问题，需要去相关主题的代码仓库去看看使用指南，是不是缺少什么依赖。
